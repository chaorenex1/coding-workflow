# 记忆数据模型

## 1. 核心概念定义

### 1.1 记忆领域（Domain）

**定义**：记忆所属的语义范畴，用于将记忆按主题大类进行聚合管理。每个领域对应一个独立的知识库，实现检索隔离。

| 领域 | 标识 | 语义含义 | 典型内容示例 |
|------|------|----------|--------------|
| 偏好 | preference | 用户的个人偏好和习惯 | "用户喜欢简洁的回复风格" |
| 事件 | event | 具体发生的事情 | "用户明天下午3点有会议" |
| 关系 | relationship | 人际关系或实体关联 | "用户的项目经理是张三" |
| 行为 | behavior | 用户的行为模式 | "用户通常在工作日早上9点上线" |
| 知识 | knowledge | 通用知识和事实 | "项目的部署流程是..." |

---

### 1.2 记忆类型（Type）

**定义**：参考认知心理学的记忆分类理论，按信息性质和持久性划分。

| 类型 | 标识 | 认知含义 | 特征 |
|------|------|----------|------|
| 感知记忆 | perceptual | 感官信息的短暂暂存 | 时效短、容量小、原始感知 |
| 情景记忆 | episodic | 特定时间和地点的经历 | 有时间线、个人化、叙事性 |
| 语义记忆 | semantic | 概念、事实和一般知识 | 抽象、去情境化、可共享 |
| 程序记忆 | procedural | 技能和操作方法 | 隐性、通过实践获得、难言传 |

---

### 1.3 记忆话题（Topic）

**定义**：LLM 从记忆内容中自动提取的主题标签，用于同主题记忆的聚合和检索。

**特性**：
- 由 LLM 智能提取，非预定义枚举
- 同一话题的记忆会自动合并（追加模式）
- 用于知识图谱中建立"共享话题"关联边

**生成流程**：输入记忆内容 → LLM 分析 → 输出话题名称 + 相关内容片段

**示例**：
- 输入："我下周要去北京出差，记得帮我订机票"
- 提取话题："出差安排"
- 内容片段：["下周去北京", "需要订机票"]

---

## 2. 兼容矩阵

### 2.1 领域 × 类型 推荐组合

| 领域 \ 类型 | perceptual | episodic | semantic | procedural |
|-------------|:----------:|:--------:|:--------:|:----------:|
| **preference** | ○ | ● | ● | ○ |
| **event** | ○ | ● | ○ | ○ |
| **relationship** | ○ | ○ | ● | ○ |
| **behavior** | ● | ● | ● | ● |
| **knowledge** | ○ | ○ | ● | ● |

**图例**：● 推荐组合 / ○ 可能但少见

### 2.2 组合语义说明

| 组合 | 语义 | 示例 |
|------|------|------|
| preference + semantic | 持久的偏好知识 | "用户偏好使用 Python 进行数据分析" |
| preference + episodic | 偏好形成的事件 | "用户上次选择了深色主题" |
| event + episodic | 具体事件记录 | "2024年3月15日参加了项目评审" |
| relationship + semantic | 稳定的关系知识 | "李四是前端团队负责人" |
| behavior + procedural | 行为习惯流程 | "用户收到bug报告后先看日志" |
| knowledge + semantic | 通用知识 | "系统的认证流程使用JWT" |
| knowledge + procedural | 操作方法 | "部署流程分为三步..." |

### 2.3 话题与领域的关系

话题是领域内的细分主题，关系如下：

```
领域（Domain）          话题（Topic）              具体记忆
    │                       │                        │
    ├── preference    ──→  "界面偏好"     ──→  记录A, 记录B, ...
    │                       ├── "沟通风格"   ──→  记录C, 记录D, ...
    │                       └── "工作习惯"   ──→  记录E, ...
    │
    ├── event        ──→  "会议安排"     ──→  记录F, 记录G, ...
    │                       ├── "出差计划"   ──→  记录H, ...
    │                       └── "里程碑"     ──→  记录I, ...
    │
    └── knowledge    ──→  "项目架构"     ──→  记录J, 记录K, ...
                            ├── "API规范"    ──→  记录L, ...
                            └── "部署流程"   ──→  记录M, ...
```

## 3. 记忆组成结构

### 3.1 完整记忆实体

一条完整的记忆由以下部分组成：

```
┌─────────────────────────────────────────────────────────┐
│                      Memory 实体                         │
├─────────────────────────────────────────────────────────┤
│  身份标识                                                 │
│  ├── ID: 全局唯一标识符                                   │
│  └── 所属领域ID: 关联的 MemoryBase                        │
├─────────────────────────────────────────────────────────┤
│  分类属性                                                 │
│  ├── 领域: preference / event / relationship / ...       │
│  ├── 类型: perceptual / episodic / semantic / procedural │
│  ├── 来源: chat / qa / agent_task / document / ...       │
│  ├── 话题: LLM 自动提取的主题                             │
│  └── 标签: 用户自定义标签列表                             │
├─────────────────────────────────────────────────────────┤
│  内容与关联                                               │
│  ├── 内容: 记忆的文本内容                                 │
│  └── 知识文档ID: 用于 RAG 检索的向量索引文档              │
├─────────────────────────────────────────────────────────┤
│  上下文                                                   │
│  ├── 用户ID: 所属用户                                    │
│  ├── 项目ID: 所属项目（可选）                            │
│  └── AgentID: 来源 Agent（可选）                         │
├─────────────────────────────────────────────────────────┤
│  时间戳                                                   │
│  ├── 创建时间                                            │
│  ├── 更新时间                                            │
│  └── 访问时间: 最后被检索的时间                           │
├─────────────────────────────────────────────────────────┤
│  生命周期                                                 │
│  ├── 质量分数: 0-1，由学习引擎更新                        │
│  ├── 访问次数: 被检索的累计次数                           │
│  ├── 生存时间: 可选的过期时间（秒）                       │
│  └── 删除标记: 软删除标志                                │
└─────────────────────────────────────────────────────────┘
```

### 3.2 领域聚合结构

多个记忆按领域聚合：

```
┌─────────────────────────────────────────────────────────┐
│                    MemoryBase（领域聚合）                │
├─────────────────────────────────────────────────────────┤
│  ID: 领域聚合唯一标识                                    │
│  用户ID: 所属用户                                        │
│  领域: preference / event / relationship / ...           │
│  类型: 该领域默认的记忆类型                               │
│  知识库ID: 关联的 RAG 知识库                             │
│  删除标记                                                │
├─────────────────────────────────────────────────────────┤
│              关联的记忆记录（1:N）                        │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                 │
│  │Memory 1 │  │Memory 2 │  │Memory 3 │  ...            │
│  │话题: A  │  │话题: B  │  │话题: A  │                 │
│  └─────────┘  └─────────┘  └─────────┘                 │
└─────────────────────────────────────────────────────────┘
```

### 3.3 标签关联结构

记忆与用户自定义标签的多对多关系：

```
┌──────────────┐      ┌──────────────────┐      ┌──────────────┐
│  Memory      │      │ TagAssociation   │      │ UserTag      │
├──────────────┤      ├──────────────────┤      ├──────────────┤
│  ID          │◄────►│  MemoryID        │      │  ID          │
│  内容        │      │  TagID           │◄────►│  名称        │
│  话题        │      │  分配来源        │      │  向量表示    │
│  ...         │      │  分配时间        │      │  父标签ID    │
└──────────────┘      │  上下文          │      │  使用次数    │
                      └──────────────────┘      └──────────────┘

分配来源: manual(手动) / auto(自动) / learned(学习获得)
```

## 4. 三者协同工作流程

### 4.1 记忆创建流程

```
用户输入内容
    │
    ▼
┌─────────────────────────────────────┐
│  LLM 分类阶段                        │
│  ├── 分析内容                        │
│  ├── 确定领域              │
│  └── 确定类型                │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  话题提取阶段                        │
│  ├── LLM 提取话题           │
│  └── 生成内容片段                    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  标签生成阶段                        │
│  ├── LLM 生成标签                    │
│  ├── 向量化标签                      │
│  └── 相似标签去重                    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  持久化阶段                          │
│  ├── 查找或创建领域聚合(MemoryBase)  │
│  ├── 查找同话题已有记忆              │
│  │   ├── 存在 → 追加内容             │
│  │   └── 不存在 → 创建新记录         │
│  ├── 写入知识库(RAG索引)             │
│  └── 更新知识图谱                    │
└─────────────────────────────────────┘
```

### 4.2 话题合并机制

当新记忆与已有记忆话题相同时，采用追加模式：

| 场景 | 处理方式 |
|------|----------|
| 新话题 | 创建新 MemoryRecord，新建知识文档 |
| 已有话题 | 复用 MemoryRecord，LLM 合并内容到知识文档 |
| 标签变化 | 更新标签列表，同步标签关联表 |

## 5. 设计决策

### 5.1 为什么需要三个维度

| 维度 | 解决的问题 |
|------|------------|
| 领域 | 粗粒度分类，支持领域隔离检索 |
| 类型 | 语义分类，支持按认知特性处理 |
| 话题 | 细粒度聚合，支持同主题记忆合并 |

### 5.2 维度之间的关系

- **领域** 决定了记忆存储在哪个知识库
- **类型** 影响记忆的检索优先级和生命周期策略
- **话题** 决定了记忆是否与已有记忆合并
